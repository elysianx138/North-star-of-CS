# DAY3 — 指针和引用的本质

> **阅读前提：** 需要基础指针概念、引用概念和基础 C++ 语法。
> **核心主题：** 指针是什么？引用又是什么？底层一样吗？

---

## 一、指针的本质

### 指针 = 存地址的变量

```cpp
int a = 42;
int* p = &a;  // p 存的是 a 的地址
```

```
内存里：
      p（栈上）             a（栈上）
    ┌────────────┐       ┌──────┐
    │   &a       │ ───→  │  42  │
    └────────────┘       └──────┘
```

- `p` → 取 p 的值，即 a 的地址
- `&p` → p 自己在栈上的地址
- `*p` → 解引用，取 a 的值

### 指针运算：`p + 1` 跳几个字节？

```cpp
int arr[4] = {10, 20, 30, 40};
int* p = arr;   // p → arr[0]

p + 1;  // 地址 + 4 字节，不是 + 1 字节！
```

**原理：**
```
地址:  1000  1004  1008  1012
      [10]  [20]  [30]  [40]
       ↑
       p
p + 1 → 1004（跳过完整的一个 int，4 字节）
p + 1 ≠ 1001（否则取到的是 10 的后半截 + 20 的前半截 = 乱码）
```

**本质：** 指针的类型决定了"如何理解内存"。`p + n` = 地址 + `n * sizeof(T)`。

### 关键发现：栈上地址顺序不一定按声明顺序

```cpp
int a = 42;
int* p = &a;
// &p 可能 > &a，也可能 < &a
```

**原因：** 编译器可以在同一个栈帧内**自由重排局部变量**。栈"向下生长"指的是**跨函数的栈帧**，不是同一函数内的局部变量。

## 二、引用的本质

### 引用 = 不能改指向的指针（自动解引用）

```cpp
int a = 42;
int& r = a;    // r 是 a 的别名
```

**对比指针：**

| 操作 | 指针 `int* p = &a` | 引用 `int& r = a` |
|:----|:------------------|:-----------------|
| 读值 | `*p` | `r` |
| 改值 | `*p = 100` | `r = 100` |
| 改指向 | `p = &b;` ✅ | `r = b;` ❌（这是赋值，不是改指向） |
| 取地址 | `&p` 得到 p 自己的地址 | `&r` 得到 a 的地址（别名透明） |

### 汇编对比（g++ -S）

**不开优化（-O0）：**
```asm
movl    $42, -28(%rbp)        # a = 42

; 指针：int* p = &a
leaq    -28(%rbp), %rax       # rax = &a
movq    %rax, -24(%rbp)       # p 在 rbp-24，占 8 字节（存的 &a）

; *p = 100
movq    -24(%rbp), %rax       # rax = p（取地址）
movl    $100, (%rax)          # *p = 100

; 引用：int& r = a
leaq    -28(%rbp), %rax       # rax = &a
movq    %rax, -16(%rbp)       # r 在 rbp-16，占 8 字节（存的 &a）

; r = 200
movq    -16(%rbp), %rax       # rax = r（取地址）
movl    $200, (%rax)          # *r = 200
```

**结论：不开优化时，指针和引用在汇编层面完全一样，都占 8 字节存地址。**

**开优化（-O2）：**
```asm
; main 函数只剩：
subq    $16, %rsp
movl    $200, 4(%rsp)         # 直接把 a 赋值为 200
; 指针 p 消失了！
; 引用 r 也消失了！
```

**结论：** 开优化后编译器看穿间接层，引用（和指针）被完全优化掉。引用 ≈ `int* const` 的语法糖，但编译器可以利用"不能改指向"的特性做更多优化。

### 关于"引用有没有地址"的讨论

| 层面 | 引用有地址吗 |
|:----|:-----------|
| C++ 语法层面 | ❌ `&r` 返回的是 `&a`，拿不到引用自己的地址 |
| 汇编层面（-O0） | ✅ 确实占 8 字节，有地址，存的是 `&a` |

**一句话：** C++ 语言让你"碰不到"引用自己，但底层它确实是个存地址的变量。

## 三、`&` 的两种身份

```cpp
int* p = &a;     // ① 取地址符（address-of）
                  //    & 在变量左边 → 取变量的地址
int& r = a;      // ② 引用声明符（reference）
                  //    & 在类型右边 → 声明引用
```

**底层一样**（都是取地址），**语法角色不同**：

| | 取地址 `&a` | 引用 `int& r` |
|:--|:----------|:------------|
| 出现位置 | 表达式 | 类型声明 |
| 含义 | "给我 a 的地址" | "r 是 a 的别名" |
| 能否独立存在 | ✅ `p = &a` | ❌ 必须绑定，`int& r;` 不合法 |

**口诀：** `&` 在**类型右边**是引用，在**变量左边**是取地址。

## 四、const 指针组合拳

四种组合，核心看 `const` 在 `*` 的左边还是右边：

### ① 指向 const int 的指针
```cpp
const int* p;   // 也可写成 int const*
```
| 操作 | 结果 |
|:----|:----|
| `p = &b;` | ✅ 可以改指向 |
| `*p = 100;` | ❌ 不能改值 |

### ② const 指针，指向 int
```cpp
int* const p = &a;  // 必须初始化！
```
| 操作 | 结果 |
|:----|:----|
| `p = &b;` | ❌ 不能改指向 |
| `*p = 100;` | ✅ 可以改值 |

### ③ 两者都 const
```cpp
const int* const p = &a;  // 必须初始化！
```
| 操作 | 结果 |
|:----|:----|
| `p = &b;` | ❌ 不能改指向 |
| `*p = 100;` | ❌ 不能改值 |

### ④ 引用 + const
```cpp
const int& r = a;  // 不能通过 r 改 a
// 引用本身就不能改指向，再加上 const 禁止改值
```

### 记忆口诀

> **const 在 `*` 左边 → 修饰指向的值（不能改 `*p`）**
> **const 在 `*` 右边 → 修饰指针本身（不能改 `p`）**

### 从右往左读声明法

```cpp
int const* p
// p 是 → *（指针）→ const → int
// "p 是一个指向 const int 的指针"

int* const p
// p 是 → const → *（指针）→ int
// "p 是一个 const 指针，指向 int"

int*& r
// r 是 → &（引用）→ *（指针）→ int
// "r 是一个指向 int 的指针的引用"
```

### 权限收窄原则

```
int* → const int*    ✅ 收窄（安全）
const int* → int*    ❌ 放宽（不安全，可能破坏 const）
```

这就是 `bar(int* const p)` 接收 `&b`（`const int*`）会编译报错的原因。

## 五、二级指针 `int**`

```
int** p（二级指针）
    ┌────────────┐
    │   &ptr     │ ──→  ptr（一级指针）
    └────────────┘         ┌────────────┐
                           │   &a       │ ──→  a（int 值）
                           └────────────┘         ┌──────┐
                                                  │  42  │
                                                  └──────┘
```

**用途：** 在函数内部修改指针本身

```cpp
void setToNull(int** p) {
    *p = nullptr;  // 修改传入的指针本身
}
int* ptr = &a;
setToNull(&ptr);  // 传指针的地址
// 现在 ptr == nullptr
```

**vs 一级指针（只能改值不能改指针本身）：**
```cpp
void trySetNull(int* p) {
    p = nullptr;   // 只改了 p 的副本
}
int* ptr = &a;
trySetNull(ptr);   // ptr 还是 &a，没变
```

## 六、指针的引用 `int*&`

```cpp
int* ptr = &a;
int*& r = ptr;  // r = ptr 的别名
r = nullptr;    // 等价于 ptr = nullptr
```

**对比二级指针：**

| 方式 | 传参 | 函数内写法 | 本质 |
|:----|:----|:----------|:----|
| `int**` | `&ptr` | `*p = ...` | 传指针地址，手动解引用 |
| `int*&` | `ptr` | `r = ...` | 传引用，自动解引用 |

**从汇编角度理解：**
- `int*& r` ≈ `int** const`（不能改 r 的绑定，但能通过 r 改 ptr）
- r → ptr → a，不允许 r → 其他变量
- 但允许 `r = &b;` → ptr 指向 b

---

## 七、指针 vs 引用：实战选用原则

| 场景 | 用指针 | 用引用 |
|:----|:------|:------|
| 需要改指向 | `p = &b;` ✅ | ❌ 不能 |
| 可以为空 | `int* p = nullptr;` ✅ | ❌ 必须初始化 |
| 操作数组/遍历 | `arr++` ✅ | ❌ |
| 函数参数减少拷贝 | `void foo(int* p)` | `void foo(int& r)` 更干净 |
| 运算符重载 | ❌ | ✅ 只能返回引用 |
| 链式调用 | ❌ 语法别扭 | ✅ `cout << a << b` |

**经验原则：能用引用就用引用，需要指针特性（改指向/可为空/数组）时才用指针。**

## 八、疑问与讨论

### Q1: 为什么指针和引用占 8 字节？不是 4 字节？
**A:** 64 位系统的地址范围是 0 ~ 2^64-1，需要 8 字节存储。32 位系统下指针是 4 字节。

### Q2: &p 为什么可能比 &a 地址小（高地址）？
**A:** 编译器可以在同一个栈帧内重排局部变量。"栈向下生长"指的是**跨函数的栈帧**关系，不保证函数内变量按声明顺序排列。

### Q3: 引用到底有没有地址？
**A:** C++ 语法层面 `&r` 返回 `&a`，你拿不到引用自己的地址。但汇编层面（-O0）引用确实占 8 字节存地址。

### Q4: int*& r 等价于 int** const 吗？
**A:** 从汇编底层看可以这么理解——r 不能改绑定，但能通过 r 修改指向的指针变量。

### Q5: 为什么要学汇编？
**A:** 不需要精通，但看懂关键几行（赋值、取地址、解引用）能帮你理解"代码到底变成了什么指令"。对于 AI Infra（性能敏感），这种底层意识很宝贵。
