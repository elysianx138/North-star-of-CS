## DAY 1 — 编译流水线 ✅

- 四阶段编译：`.cpp → .i → .s → .o → exe`

- C++ 没有"运行时解释"的开销
- 多文件分步编译 + 链接的意义（增量编译）
- 动手实验：改汇编改输出、验证跨平台

## DAY 2 — 内存布局 ✅
- 程序的内存地图：栈 / 堆 / 数据段 / 代码段
- 指针本质：栈上的变量，存着堆的地址（`p` vs `&p` vs `*p`）
- 函数调用栈帧：叠盘子理论（返回地址 + 旧 rbp + 局部变量）
- 栈 vs 堆性能对比：一次大分配 2x 差距，反复小分配 16x 差距
- BSS 区精髓：文件不占空间，运行时照样占内存
- 汇编验证：`subq %rsp` 一条指令 vs `call _Znwm` 一堆指令

## DAY 3 — 指针 & 引用 ✅
- 指针本质：存地址的变量
- 指针运算：`p + n` = 跳 `n * sizeof(T)` 字节
- 引用本质：不能改指向的指针，自动解引用
- 汇编对比（-O0 vs -O2）：不开优化引用占 8 字节存地址，开优化消失
- `&` 的两种身份：取地址符（表达式）vs 引用声明符（类型）
- const 指针组合拳：`const int*` vs `int* const`
- 二级指针 `int**` & 指针引用 `int*&`
- 指针 vs 引用实战选用原则

## DAY 4 — 数组 & 动态内存管理 🔲

- `new`/`delete` 的底层机制
- `malloc/free` vs `new/delete`
- 野指针 / 悬空指针 / 内存泄漏
- `new[]`/`delete[]` 数组版本
