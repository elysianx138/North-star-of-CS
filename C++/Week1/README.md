# Week 1：内存与编译——C++ 的"为什么"

> **定位：** 语法够用，专攻原理和工程细节
> **背景：** 牛客竞赛分 800，做题量 100~200，ACM 入门水平

## 每日安排

| 天数 | 主题 | 解决的问题 |
|:---|:-----|:---------|
| **Day 1 ✅** | 编译流水线 | .cpp → .i → .s → .o → 可执行文件，C++ 和 Python 的本质区别 |
| **Day 2** | 内存布局 — 栈 vs 堆 vs 代码段 | 变量存在哪？int a = 42 的 42 到底放在哪块内存？ |
| **Day 3** | 指针和引用的本质 + & 的两种身份 | 取地址 vs 引用，\* 的三种含义（乘/声明指针/解引用） |
| **Day 4** | const 全部用法 | const int* vs int* const，const 引用，const 成员函数 |
| **Day 5** | #include 机制 + 头文件设计原则 | "" vs <>，头文件守卫，声明和定义为什么分开 |
| **Day 6** | 初始化大乱斗 | () vs {} vs = 的区别，默认初始化/值初始化/列表初始化 |
| **Day 7** | 复盘 + 选一个概念深挖 | 你最模糊的那个，搞到透 |

## 教学原则

- 不讲语法（你有了），只讲原理和工程
- 你敲代码，我做引导
- 每个概念先和 Python 对比
- 每个概念和 AI Infra（KVCache、推理加速）建立联系
- 不赶进度，每天 1 小时
